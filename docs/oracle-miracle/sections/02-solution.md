# 2. 실현가능성 (Solution)

## 2-1. 창업아이템 현황 (준비 정도)

### □ IDLE Forge 플랫폼 구성과 단계

IDLE Forge는 세 레이어로 구성된 통합 플랫폼이며, 일부는 PoC 단계, 일부는 4년 검증 단계로 본 사업에서 통합 플랫폼으로 완성합니다.

| 레이어 | 구성 | 현재 단계 |
|---|---|---|
| 콘텐츠 양산 | 자연어→게임 데이터 변환 워크플로우, 도메인 특화 생성 스킬 30+종, 스키마·밸런스 검증 에이전트, AI 아트(Midjourney·LayerAI) | PoC 및 검증, 사내 라이브 게임에 활용 중 |
| 엔진 | 유니티 & .NET 기반 실시간 네트워크 게임 엔진 idlez, 글로벌 멀티 리전·콘텐츠 핫 패치 | 4년 운영 검증, 라이브 게임 2종 가동 |
| 라이브 운영 | 콘텐츠 컴파일·검증·배포 파이프라인, KPI 모니터링 | 사내 스크립트 → 본 사업에서 OCI Functions 완전 자동화 |

OCI 네이티브 전환은 협약 개시 후 본격 착수('26.04~)하며, 검증 사례 게임 닌자 서바이벌 2는 컨셉·코어 루프 설계 완료, MVP 제작 직전 단계입니다. 자체 라이브 게임 2종은 3인 인력의 AI 기반 운영으로 월 약 5,000만원 매출이 발생하고 있으며, 이 매출이 자기부담사업비 재원이자 1-2의 D7 +5%p 실증 데이터와 함께 본 사업의 1차 검증 근거입니다.

> **[도식 삽입 — 그림 4]** IDLE Forge 3-레이어 아키텍처
> 설명: 콘텐츠 양산 → 엔진 → 라이브 운영 데이터 흐름과 OCI 결합 지점.

## 2-2. 창업아이템 실현 및 구체화 방안

### □ 협약기간 내 협업 머신 3종 개발

  - **Trigger AI 협업 머신** — 이벤트 트리거(스폰·사망·영역 진입)와 NPC·몬스터 행동 패턴을 자연어 정의로 받아 검증 가능한 로직·데이터로 변환. 4X 연맹 NPC, 서바이벌 보스, 하우징 NPC 이벤트 등에 표준 컴포넌트로 적용.
  - **맵·레벨 에디터 협업 머신** — 게임 맵(지형·오브젝트·이벤트 포인트) 초안을 자연어로 생성하고 디자이너가 후처리·확정. 하우징 영지·서바이벌 스테이지 양산에 직접 활용.
  - **다중 게임 모드 지원 패키지** — 서바이벌·하우징·4X·방치형 모드별 콘텐츠 양산 워크플로우 패키지화. B2B 공급 시 라이선시가 모드에 맞게 선택·조합.

LLM 운영은 멀티 클라우드 구조입니다. 콘텐츠 생성의 창의·도메인 톤은 외부 LLM(Claude·GPT 등)을 유지하고, 반복 검증·RAG 일관성·게임 데이터 임베딩 검색은 OCI Generative AI Service·Agents·Oracle AI Database/Vector Search로 흡수합니다.

> **[도식 삽입 — 그림 5]** IDLE Forge 협업 머신 3종
> 설명: Trigger·맵·다중 모드의 입력(자연어)·출력(검증된 로직·데이터)·후처리.

### □ 에셋 양산 자동화 — 콘텐츠 양·깊이 장벽 완화

4X 시장 진입 장벽인 콘텐츠 양·깊이를 10인 이하 조직에서 처리하기 위해 에셋 제작 전 단계를 AI 양산 + 클라우드 자동화로 통합합니다. 전통 게임사 15~20인 콘텐츠팀의 일부 반복 업무를 당사 3~5인 + AI 워크플로우로 흡수하는 구조입니다.

| 카테고리 | 월 양산 목표 | 전통 환산 | 워크플로우 |
|---|---|---|---|
| 비주얼(아이콘·UI·배너) | 100~150개 | 4~5인 → 1인 | Midjourney·LayerAI + 닌자 IP LoRA, 다단계 에이전트 IP·라이선스 검증, 임베딩 중복 탐지, 다중 해상도 변환 |
| 배경·환경 | 15~25개 | 2~3인 → 0.5인 | 비주얼 워크플로우 공통 |
| 음향·BGM·SFX | BGM 5~8 + SFX 50~80 | 1~2인 → 0.3인 | Suno·Udio·ElevenLabs, 닌자 IP 프롬프트 세트, 음량·라이선스 검사 |
| 인게임 텍스트·다국어 | 300~500개 + 5~7개 언어 | 2인 + 외주 → 0.5인 + AI | 외부 LLM 1차 생성 + OCI Language 번역 API + RAG 기반 사내 용어집(닌자 IP 용어 200~500) |
| 2D 애니메이션·이펙트 | 10~15개 | 1~2인 → 0.5인 | 키 프레임 + 프레임 보간 AI, 모션 라이브러리 표준화 |

기획자가 "연맹 시즌 2 콘텐츠 패키지"를 정의하면 비주얼·음향·텍스트·애니메이션 초안 생성·검수·CDN 배포까지 자동 진행됩니다. 위 자동화의 OCI 서비스 매핑(Generative AI Agents·Oracle AI Database/Vector Search·Object Storage·Functions·Language·Data Science) 상세는 5장에서 다룹니다.

> **[도식 삽입 — 그림 6]** 에셋 제작 워크플로우와 OCI 통합

### □ 운영·배포 — 10인 이하 글로벌 라이브 운영

  - **엔진 멀티 리전** — idlez 서버(.NET 백엔드)를 컨테이너 환경에서 한·일·중·동남아 운용, 서버리스 자동 스케일로 4X 연맹 전쟁 폭증 트래픽 대응
  - **콘텐츠 핫 패치** — 콘텐츠 정의 푸시 시 빌드·검증·CDN 배포 무인 진행
  - **다국어 CS 자동화** — 5~7개 언어 자동 응대, 닌자 IP 고유 용어 일관 유지
  - **운영 KPI 인텔리전스** — 분산 트레이싱·이상 패턴 탐지·LTV 예측으로 라이브 사고·결제 변동 자동 감지
  - **인게임 AI 모듈** — 동적 NPC·이벤트 표준 컴포넌트, 4X 외교·하우징 NPC·시즌 이벤트에 활용

> **[도식 삽입 — 그림 7]** OCI 기반 운영·배포 아키텍처
> 설명: 모바일→CDN→OKE(Seoul·Chuncheon, idlez 서버)→Object Storage·DB + 운영 자동화 레이어와 Osaka GenAI Agents 분리.

IDLE Forge의 모든 레이어를 한 게임에서 통과시키는 첫 통합 실증입니다. 협약 종료 시점 글로벌 베타 출시를 목표로 하며, 컨셉·BM·일정·매출은 3장에서 다룹니다.

> 설명: 서바이벌(유입) → 하우징(체류) → 4X(결제·LTV)의 단계별 깔때기와 KPI.

### □ 차별성과 경쟁력

  ○ 수직 통합 카테고리 — AI 콘텐츠 양산·자체 게임 엔진·라이브 운영을 한 코드베이스에서 결합한 사례는 시장에 없습니다. **UGC 플랫폼(Roblox·Core)은 모바일 4X·서바이벌 복합 시스템에 부적합하고, AI 게임 생성 플랫폼(Rosebud·Astrocade)은 라이브 운영·BM·다국어 미통합, 게임 백엔드 SaaS(Pragma·AccelByte)는 AI 양산·자체 엔진 부재**. IDLE Forge는 이들의 합집합 영역에 단독 진입하며, **자체 라이브 게임 운영 데이터로 플랫폼이 실시간 검증**되는 점이 외부 게임 의존형 일반 AI 도구 회사 대비 결정적 차이입니다.

  ○ 라이브 게임 기반 실시간 검증 — 자체 라이브 게임 2종(누적 170만 DL, 월 5,000만원)이 플랫폼의 1차 검증 환경입니다. 2025년 신규 모드 추가 시 D7 리텐션 +5%p·ARPDAU +5%(p<0.05) 실증.

  ○ 검증된 팀의 사내 보유 — idlez 엔진을 직접 작성한 CTO와 닌자 IP 라이브 전 기간 참여 아트 2인이 사내에 재직 중입니다. 다년간 함께 일한 합 맞춤 상태로 협약 개시 즉시 본격 가동합니다.

### □ 기술 보호 계획

통합 워크플로우·도메인 노하우 영역이라 특허만으로는 어렵고 영업비밀·저작권·라이선스 계약·기술적 분리를 결합합니다.

| 보호 대상 | 보호 방식 | 핵심 조치 |
|---|---|---|
| AI 콘텐츠 자동 생성·검증 워크플로우 | 특허 출원 | 협약기간 내 1~2건 변리사 의뢰 출원 |
| idlez 엔진 핵심 로직 (실시간 네트워크·핫 패치) | 저작권 등록 + 영업비밀 | 컴퓨터프로그램 저작권 등록, 사내 접근 권한 단계 관리, NDA 의무화 |
| AI 콘텐츠 생성 스킬·검증 에이전트 | 저작권 등록 + 라이선스 계약 | 도메인 특화 스킬 라이브러리 저작권 등록, 외부 공급 시 인터페이스만 공개 |
| 파인튜닝 도메인 모델 (OCI Model Import) | 영업비밀 + 인프라 격리 | OCI 폐쇄형 환경, 모델 가중치 외부 노출 차단 |
| 닌자 IP (캐릭터·세계관·게임 디자인) | 저작권 + 상표권 | 당사 소유 관리, 글로벌 주요 시장 상표 등록 검토 |
| IDLE Forge 브랜드 | 상표권 | 협약기간 내 국내 상표 출원, 해외 확장은 협약 종료 후 |

특허 출원 후보는 「자연어 입력 기반 게임 콘텐츠 자동 생성·검증 시스템 및 방법」, 「외부 LLM과 게임 엔진을 결합한 콘텐츠 핫 패치 파이프라인」 2건입니다. B2B 공급 단계에서는 핵심 로직을 OCI 환경 내에서만 실행하고 라이선시는 결과물(생성 게임 데이터)에만 접근하며, 표준 라이선스 계약에 역공학·재배포 금지·사용 범위 제한·감사권을 포함합니다.
