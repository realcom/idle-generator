# 정부지원 사업계획서 본문 작성

> source/에 정리된 사실을 평가자가 읽을 본문 단락으로 변환한다. 톤은 한국 정부지원사업 실제 합격 문서를 기준으로 잡는다.

## 작성 단계

1. **섹션 양식 파악**: 해당 섹션의 PDF 양식 안내문구(파란색 ※)를 정확히 읽고, 평가자가 그 항목에서 무엇을 보는지 확정.
2. **source 사실 추출**: 인자로 주어진 source 파일에서 본 섹션과 연결되는 사실 골라내기. 사실은 변형 금지 (수치·고유명사 그대로).
3. **단락 초안**: 아래 톤 원칙·금지 패턴 모두 적용해 작성.
4. **자가 검토**: 작성한 단락을 "AI 티 체크리스트"로 다시 훑어 5개 이상 걸리면 재작성.
5. **저장**: `docs/<프로젝트>/sections/<섹션>.md`에 저장.

## 평가자가 보는 것 (PSST 가중치)

서류 단계 가중치 분포:
- **문제인식 + 실현가능성**: 약 60% — 가장 중요. 객관 근거·기술 차별성·시장 데이터 핵심.
- **성장전략**: 약 25% — BM·경쟁사·확장 시나리오.
- **팀 구성**: 약 15% — 역량·트랙 레코드.

요약(개요) 부분은 별도로 평가에 절대적 영향. 본문보다 먼저 읽힘.

## 톤 원칙 (반드시 지킬 것)

### 종결·문체

- **"~합니다 / ~입니다 / ~예정입니다 / ~확보하였습니다"** 단정형 평어체 기본. 약속·계획은 미래형, 확정·이력은 과거·현재완료형.
- 명사형 종결("~함, ~임")은 표·목록·요약에서만. 본문 단락은 평어체 권장 (캠퍼스타운 합격 문서 톤 기준).
- 능동태 우선. "구현됩니다" → "구현합니다".

### 구조

- 한 단락 2~4줄. 5줄 넘어가면 단락 쪼개기.
- 헤더 단계는 `□` (큰) → `○` (중간) → `-` (글머리) 또는 PDF 양식의 `1-1 / 1-1-1` 번호 그대로.
- 한 글머리(`-`) 안에는 한 주장 + 그 근거 한두 가지. 글머리 중첩 피함.

### 근거·수치

- 단정 옆에는 근거 한 줄: "2025년 ○○사 출시작 3개월 500만 DL 돌파" 같은 외부 인용 또는 자사 실측치.
- 퍼센트·금액·기간은 **반올림한 자연수**로. "5x+", "70%↓" 같은 단축 기호 금지 → "5배 이상", "70% 절감".
- 약어는 첫 등장 시 풀어쓰고 괄호: "LTV(유저 생애가치)", "CPI(설치당 비용)".

### 자기 회사 호칭

- 본문에서 자기 회사는 **"당사" 또는 "본 회사"** (또는 회사명 그대로). "우리"는 가급적 피함.
- "본 사업", "본 아이템"은 첫 단락에서 한 번 정도까지. 매 단락 반복 X.

## 금지 패턴 (AI 티 체크리스트)

작성 후 아래 항목 셀프 체크. **5개 이상 걸리면 재작성**.

1. ⬜ **대칭 강박 구조** — "4대 핵심성과 + 4대 전략", "3단계 + 3단계" 같은 균등 대칭. 평가자는 자연스러운 비대칭(예: 5-3-2)을 신뢰함.
2. ⬜ **메타 표현** — "본 사업의 핵심 명제", "앞서 언급했듯이", "한 줄 요약", "TL;DR".
3. ⬜ **과한 정확성 어휘** — "5x+", "70%↓", "이상", "이내" 같은 단축 기호·범위 표시 남발. 단정값으로.
4. ⬜ **마케팅 풍 형용사 반복** — "혁신적", "차별화된", "획기적", "압도적", "전례 없는". 한 단락에 한 개로 제한.
5. ⬜ **이모지·기호** — ✓ ★ 🚀 → 모두 제거. 표 안의 "✓"는 "○" 또는 "있음/없음"으로.
6. ⬜ **굵게 강조 일률화** — 단락마다 굵은 글자 한 줄씩 박는 패턴. 핵심 한두 군데만.
7. ⬜ **모든 글머리 같은 길이** — 글머리는 자연스러운 길이 차이를 가져야 함.
8. ⬜ **상투적 헤더** — "한 줄 요약", "핵심 메시지", "TL;DR", "전략적 시사점".
9. ⬜ **추상 동사 남발** — "활용한다, 구현한다, 최적화한다, 도모한다" 한 단락 3회 이상. 구체 동사로 풀어쓰기 ("OCI Functions로 빌드 → 배포까지 자동 수행").
10. ⬜ **표 중첩** — 표 안에 표, 셀 안에 글머리 다단. 깊이 1단 유지.
11. ⬜ **양극 비교 표** — "Before/After", "전통적 방식 vs 우리". 평가자는 자기 비교를 신뢰하지 않음. 외부 시장 평균과의 비교로 전환.
12. ⬜ **"~할 수 있습니다" 남발** — 가능성 표현 한 단락 2회 이상이면 가능성이 아닌 계획·이행 형태로 단정.
13. ⬜ **협력사 일변도 톤** — 글로벌 기업 협업 사업계획서에서 협력사명(Oracle, AWS 등)이 매 단락 등장하거나, "협력사 입장에서 win-win", "협력사 생태계 기여", "협력사 진출 레퍼런스" 같은 표현이 보이면 자사 사업 관점이 흐려진 것. 협력사는 사용하는 도구의 위치로 유지하고, 주어는 항상 당사·당사 사업·당사 매출.
14. ⬜ **과한 야망 어휘** — "최초 사례", "표준 제시", "선도", "전환을 이끈다" 등. 협약기간 내 실증 가능한 수준의 어휘로 절제. 평가자는 거대 비전보다 8개월 안에 입증 가능한 성과를 본다.

## 캠퍼스타운 합격 문서에서 본 자연 톤 예시

좋은 예 (실제 합격 문서):

```
○ 4X 전략 장르의 독보적 글로벌 성장세
  - 2024~2025년 모바일 시장은 4X 전략이 주도했습니다. <라스트워>, <화이트아웃 서바이벌>에
    이어 2025년 킹샷(King Shot)이 유례없는 매출 성과를 거두며 글로벌 시장의 랭크 10위에
    안정적으로 랭크한 사례는, 유입된 캐주얼 유저를 고액 결제로 전환하는 4X 시스템이 재미와
    비즈니스 모델 두가지를 동시에 잡고 있음을 시사합니다.
```

특징:
- ○ 헤더 = 명사구로 주장 압축 ("독보적 글로벌 성장세")
- 본문 첫 문장 = 단정 주장 ("4X 전략이 주도했습니다")
- 둘째 문장 = 구체 사례 3건 → 한 문장 안에서 결론까지 연결
- 약어 풀어쓰기 ("LTV(생애 가치)")
- 마케팅 형용사("유례없는") 한 번만

나쁜 예 (AI 티):

```
□ 본 사업의 핵심 명제: "IDLE Forge는 OCI 위에서 가장 큰 가치를 만든다"

본 사업의 창업아이템 IDLE Forge는 ① AI 콘텐츠 생성 하네스, ② 검증된 모바일
게임 엔진(idlez), ③ 라이브 운영 파이프라인의 수직 통합 플랫폼이며, 세 레이어
모두가 OCI 위에서 가장 큰 시너지를 낸다. 이는 단순 호스팅이 아닌 AI 워크플로우
전반의 OCI 네이티브 전환으로, 미라클 4기 가이드라인 ("OCI의 어떤 서비스를
이용하여 어떤 가치를 만들 계획이다")에 정면으로 부합한다.
```

문제:
- 메타 표현 ("본 사업의 핵심 명제")
- 따옴표로 명제 박기 (자기 인용)
- 숫자 마커 ①②③ + "수직 통합" 같은 추상 명사 연쇄
- 한 문장이 너무 길고 정보가 압축돼서 평가자가 한 번에 못 읽음
- 평어체 ("부합한다") — PSST 사업계획서는 "~합니다" 권장

재작성:

```
□ OCI와 결합되는 IDLE Forge 플랫폼 구조

당사의 IDLE Forge는 AI 콘텐츠 생성 도구·자체 모바일 게임 엔진·라이브 운영
파이프라인이 결합된 통합 플랫폼입니다. 이 세 영역 모두 OCI 서비스를 핵심
구성요소로 사용하는 구조이며, 단순히 클라우드 호스팅에 의존하는 것이 아니라
AI 워크플로우 전반을 OCI 환경에서 운영합니다.

본 구조는 미라클 4기 모집공고의 가이드라인과 부합합니다.
```

차이:
- 명사구 헤더 ("OCI와 결합되는 ~ 구조")
- 자기 인용 제거
- 정보를 두 단락으로 분리
- "~합니다" 평어체
- 따옴표·명제 박기 제거

## PSST 구조별 작성 패턴

### 1. 문제인식 (Problem)

- **1-1 배경 및 필요성**:
  - 외적 동기 (시장·기술 트렌드, 구체 사례 2~3건 인용)
  - 내적 동기 (대표·팀 경험·문제 발견)
  - 문제 정의 (객관 근거 — 데이터·논문·실측치)
- **1-2 목표시장**:
  - 시장 규모 (기관 인용)
  - 세분 시장 정의
  - 경쟁 강도 (경쟁사 N개 비교)
  - 고객 페인 포인트 (실측 데이터)

### 2. 실현가능성 (Solution)

- **2-1 현황 (준비 정도)**:
  - 신청 전까지 추진 이력 (시기·산출물)
  - 현재 시점의 TRL/MVP 수준
  - 기 보유 자산·이력 (시제품·특허·매출 등)
- **2-2 실현 및 구체화 방안**:
  - 핵심 기능·성능 구체화 방안
  - 자사 보유 역량 기반의 차별성·경쟁력 확보 방법
  - 기술 보호 계획 (특허·영업비밀 등)

### 3. 성장전략 (Scale-up)

- **3-1 BM·추진성과**: 수익 모델, 현재 진입 시장 성과 (매출표)
- **3-2 사업화 전략**: 시장 확장·고객 확보·수익 확대
- **3-3 일정·자금**: 협약기간 마일스톤, 자금 집행, 추가 조달

### 4. 팀 구성 (Team)

- **4-1 보유 역량**: 대표·팀원·장비·외부 협력 (마스킹 적용)
- **4-2 ESG**: 환경·사회·지배구조 (인디 기업 적정 수준)

## 적용 절차 (이 skill 호출 시)

1. 인자로 받은 섹션 id (예: `5-2`) 확인.
2. PDF 양식에서 해당 섹션 안내문구 다시 읽기.
3. 인자로 받은 source 파일들 Read.
4. 위 톤 원칙 + 금지 패턴 체크리스트를 머릿속에 띄우고 초안 작성.
5. 작성 후 체크리스트로 다시 훑어 5개 이상 걸리면 재작성.
6. `docs/<프로젝트>/sections/<섹션>.md`에 저장.
